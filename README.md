# colorscheme-template

A Neovim colorscheme template. Define ~30 hex colors once, and they propagate across 70+ Neovim plugin highlight groups and 45+ external app export formats.

## Quick start

### 1. Scaffold your colorscheme

```bash
./scripts/scaffold
```

You'll be prompted for a name (e.g. `nightfox`, `Ice Cream`). The script will:

- Copy the template into a new directory (lowercased, no spaces — e.g. `./icecream/`)
- Rename all files, directories, and internal references from `mytheme` to your scheme ID
- Set the display name in the extras config
- Initialize a fresh git repo

### 2. Define your palette

Open `lua/<yourscheme>/colors/dark.lua` and replace the placeholder `#000000` values with your colors. The file is organized into sections:

| Section              | Colors                                                                                            | Used for                                              |
| -------------------- | ------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| **Backgrounds**      | `bg`, `bg_dark`, `bg_darker`, `bg_highlight`                                                      | Editor, popups, sidebars, cursorline                  |
| **Foregrounds**      | `fg`, `fg_dim`, `fg_gutter`, `fg_muted`, `fg_secondary`, `fg_ghost`, `comment`                    | Text, line numbers, ghost text, comments              |
| **Core syntax**      | `blue`, `green`, `red`, `orange`, `yellow`, `magenta`, `purple`, `teal`                           | Functions, strings, errors, constants, warnings, etc. |
| **Extended palette** | `blue_dim`, `blue_bright`, `cyan`, `cyan_bright`, `green_bright`, `teal_dim`, `red_dim`, `accent` | Diagnostics, diffs, search, navigation labels         |
| **Git**              | `git.add`, `git.change`, `git.delete`                                                             | Gutter indicators                                     |

See `COLORS.md` for detailed descriptions of where every color appears.

The **light variant** is auto-generated by inverting your dark palette through the HSLuv color space. You can adjust the vibrancy with the `day_brightness` config option (0–1), or override individual colors in `lua/<yourscheme>/colors/light.lua`.

### 3. Install as a Neovim plugin

Add your scheme's directory (or git repo) to your plugin manager. For example, with [lazy.nvim](https://github.com/folke/lazy.nvim):

```lua
{
  "you/yourscheme",
  lazy = false,
  priority = 1000,
  config = function()
    require("yourscheme").setup({
      -- your options here
    })
    vim.cmd.colorscheme("yourscheme")
  end,
}
```

## Configuration

Pass options to `setup()` before loading the colorscheme:

```lua
require("yourscheme").setup({
  style = "dark",              -- "dark" or "light"
  transparent = false,         -- disable background color for terminal transparency
  terminal_colors = true,      -- set :terminal ANSI palette
  styles = {
    comments = { italic = true },
    keywords = { italic = true },
    functions = {},
    variables = {},
    sidebars = "dark",         -- "dark", "transparent", or "normal"
    floats = "dark",           -- "dark", "transparent", or "normal"
  },
  day_brightness = 0.3,        -- light mode vibrancy (0 = dull, 1 = vibrant)
  dim_inactive = false,        -- dim inactive windows
  lualine_bold = false,        -- bold lualine section headers
  cache = true,                -- cache compiled highlights for performance

  plugins = {
    all = true,                -- enable all plugin groups (auto-detected with lazy.nvim)
    auto = true,               -- auto-enable groups for loaded plugins
  },

  -- Override specific palette colors
  on_colors = function(colors)
    colors.accent = "#ff9e64"
  end,

  -- Override specific highlight groups
  on_highlights = function(hl, colors)
    hl.Comment = { fg = colors.comment, italic = false }
  end,
})
```

### Variants

Three colorscheme commands are available:

```vim
:colorscheme yourscheme        " uses the configured style (default: dark)
:colorscheme yourscheme-dark
:colorscheme yourscheme-light
```

## Plugin support

Highlight groups are included for 70+ plugins, including:

- **Completion:** blink.cmp, nvim-cmp
- **Diagnostics:** Trouble, ALE
- **File explorers:** Neo-tree, nvim-tree, Oil
- **Fuzzy finders:** Telescope, FZF
- **Git:** Gitsigns, GitGutter, Fugitive
- **Navigation:** Flash, Hop, Leap
- **Statuslines:** Lualine, Lightline, Barbecue
- **Treesitter:** syntax groups, context, rainbow delimiters
- **AI:** Copilot, Codeium, Supermaven
- **UI:** Noice, Dashboard, Barbar, Bufferline, Indent-blankline, Which-key, Mini.nvim

Plugins are auto-detected when using lazy.nvim. To manually control which groups are loaded:

```lua
require("yourscheme").setup({
  plugins = {
    all = false,
    telescope = true,
    gitsigns = true,
  },
})
```

## Extras (external app themes)

Export your palette to 45+ external applications by running the build script inside Neovim:

```bash
./scripts/build
```

This generates theme files for:

**Terminals:** Alacritty, Kitty, WezTerm, Ghostty, Foot, iTerm, GNOME Terminal, Windows Terminal, Termux, Konsole, and more

**Editors:** Sublime Text, Helix, Vim

**Tools:** Tmux, Zellij, Delta, Lazygit, Gitui, FZF, Yazi, Zathura, Btop, Dunst

**Chat/Web:** Discord, Slack, Prism.js, Tailwind CSS v4, Vivaldi

Generated files are written to `extras/`.

## Project structure

```
lua/<yourscheme>/
├── colors/
│   ├── dark.lua         # ← your palette (edit this)
│   ├── light.lua        # light overrides (optional)
│   └── init.lua         # derives composite colors from palette
├── groups/              # 70+ highlight group modules
│   ├── base.lua         # core Neovim highlights
│   ├── treesitter.lua   # syntax highlighting
│   ├── kinds.lua        # LSP completion icons
│   └── ...              # one file per plugin
├── extra/               # 45+ export generators
├── config.lua           # configuration & defaults
├── theme.lua            # applies highlights to Neovim
├── util.lua             # color math (blend, brighten, invert)
└── init.lua             # main entry point
```

## Adding highlight groups

To add support for a new plugin, create a file in `lua/<yourscheme>/groups/`:

```lua
local M = {}

---@param colors ColorScheme
function M.get(colors, opts)
  return {
    PluginHighlight = { fg = colors.blue, bg = colors.bg_dark },
  }
end

return M
```

Then register it in `lua/<yourscheme>/groups/init.lua`.
